# 页面流程设计文档

## 一、应用状态流转

### 1.1 状态图
```
[开发者原始态]
    ↓ (首次配置)
[管理员配置态]
    ↓ (配置完成并锁定)
[成员使用态]
    ↓ (可返回管理员态，需验证)
[管理员配置态]
```

### 1.2 状态检测逻辑
```python
def detect_app_state():
    if not exists("config/admin_config.json"):
        return "developer"  # 开发者态
    elif not is_locked("config/admin_config.json"):
        return "admin"      # 管理员态（未锁定）
    else:
        return "member"    # 成员态（已锁定）
```

## 二、页面导航结构

### 2.1 导航树
```
MainWindow
├── 开发者态
│   └── AdminHomePage (首次配置)
│
├── 管理员态
│   ├── AdminHomePage (配置/修改)
│   └── MemberHomePage (预览成员端)
│
└── 成员态
    ├── MemberHomePage (首页)
    ├── ListPage (模板列表)
    ├── TemplatePage (模板填写)
    └── ExportDialog (导出)
```

## 三、详细页面流程

### 3.1 开发者态流程

#### 流程 1：首次启动（开发者态）
```
启动应用
  ↓
检测到无 admin_config.json
  ↓
显示欢迎页面
  ↓
提示："请先进行管理员配置"
  ↓
自动跳转到 AdminHomePage
```

#### 流程 3：管理员修改配置（已锁定）
```
成员态下
  ↓
点击"管理员设置"（需要密码验证）
  ↓
输入管理员密码
  ↓
验证通过 → 解锁配置
  ↓

  ↓
修改配置
  ↓
保存并重新锁定
```

### 3.3 成员态流程

#### 流程 4：成员使用主流程
```
启动应用（成员态）
  ↓
进入 MemberHomePage（首页）
  ↓
填写/修改基本信息
  ↓
[保存] → 自动保存
  ↓
```

## 四、交互细节设计

### 4.1 自动保存机制
- **触发时机**：
  - 字段失去焦点时
  - 定时保存（每 30 秒）
  - 切换页面时
- **保存反馈**：
  - 状态栏显示"已保存"
  - 保存失败时显示错误提示

### 4.2 数据验证反馈
- **实时验证**：
  - 字段输入时实时验证
  - 显示验证错误提示
- **保存时验证**：
  - 所有必填字段验证
  - 格式验证
  - 逻辑关系验证
- **验证结果展示**：
  - 字段旁显示 ✓ 或 ✗
  - 错误信息提示

### 4.3 页面切换确认
- **有未保存数据时**：
  - 提示"有未保存的更改，是否保存？"
  - 选项：[保存并离开] [不保存离开] [取消]

### 4.4 数据引用显示
- **基础信息引用**：
  - 在模板页面以只读方式显示
  - 显示数据来源（"来自首页"）
  - 点击"修改"可返回首页

## 五、错误处理流程

### 5.1 数据加载失败
```
检测到配置文件损坏
  ↓
显示错误提示
  ↓
提供选项：
  - [从备份恢复]
  - [重新配置]
  - [退出程序]
```

### 5.2 Word 生成失败
```
Word 生成过程中出错
  ↓
显示错误详情
  ↓
提供选项：
  - [重试]
  - [检查数据完整性]
  - [取消]
```

### 5.3 数据验证失败
```
保存时发现验证错误
  ↓
高亮显示错误字段
  ↓
显示错误列表
  ↓
用户修正后重新保存
```

## 六、用户体验优化

### 6.1 进度提示
- Word 生成时显示进度条
- 长时间操作显示加载动画

### 6.2 快捷键支持
- `Ctrl+S`：保存
- `Ctrl+N`：新建/返回首页
- `Ctrl+E`：导出
- `Esc`：取消/返回

### 6.3 数据提示
- 字段提示信息（Tooltip）
- 示例数据展示
- 格式要求说明

### 6.4 历史记录
- 显示最近导出的文档
- 快速重新导出
