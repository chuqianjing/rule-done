## 快速入门（5分钟上手）

1. **安装应用**

   - 前往 [Releases 页面](../../releases) ，根据操作系统下载最新版本
   - 解压后双击运行即可

2. **首次启动时选择身份**

   - 选择「党支部管理员」：配置支部信息、模板通用字段
   - 选择「发展成员」：填写个人信息、生成材料

3. **基本工作流程（党员档案归档流程）**

   - **填写并发布配置**：管理员在工具上配置当前批次发展成员待归档材料的信息，并将其上传至云端的公开存储仓库；
   - **双方交流**：管理员通过群聊等方式通知当前批次成员完善待归档材料，说明纸质版材料提交截止日期及若干注意事项；
   - **获取并完善材料**：成员在工具上自动拉取到支部最新配置，然后完善待归档材料，导出文档、微调格式、打印为纸质版；
   - **归档存档**：成员将材料交至管理员处，检查无误后归档，同时在工具中锁定材料并留存档案实物图片。

## 党支部管理员操作指南

### 填写配置

#### 1. 填写支部基本信息

进入「基本信息」页，根据页面内容和实际情况填写。点击「保存」按钮保存配置。

>注：为便于管理员将更多发展党员工作的重要信息统一汇总，本工具在当前页面设置了多种板块，管理员可按需使用。

#### 2. 配置模板通用字段

进入「模板列表」页，点击需要配置的模板，进入「模板详情」页配置当前发展批次成员在该模板的通用字段。

这些通用字段会自动应用到目标成员生成的材料中，例如：
- 支委会意见
- 投票人数
- 日期落款

本工具为模板字段的配置提供了如下三种方式，管理员可按需使用：
- 锁定：勾选锁定框，管理员设定的该字段值在成员端会固定显示（不论该字段是否为空值），成员无法修改
- 提示：管理员为该字段填写相应的值，但不勾选锁定框，该字段值在成员端会以提示的方式呈现，成员可根据需要修改其值
- 无：管理员不配置该字段（既不填写值、也不勾选锁定框），该字段在成员端无任何配置信息，成员根据个人情况来填写

此外，针对需要手写的材料，本工具在其「模板详情」页为管理员提供填写要求的录入板块。

>注：为更好提升工作效率，针对同批次成员的当前阶段各项流程，建议党支部管理员统筹安排、积极联系上下级组织，使每项程序仅进行一次，从而使特定模板通用字段可更多地适用于当前批次成员。

### 发布配置

在「通用设置」页的「云端发布」分组，选择同步目标并配置相关参数，点击「立即同步到远程」即可保存相关参数并发布配置到远程云存储仓库。

本工具提供了两种云存储方式，管理员可根据自身情况选择：

- GitHub：使用说明和参数配置请见？？？
- 阿里云OSS：使用说明和参数配置请见？？？
  
仓库配置操作仅需在首次发布时进行一次，工具会保存首次发布时的参数，此后若不变更同步目标则不需要频繁设置。

>注：请先在官网配置好远程仓库，确定待存储文件 `admin_config.json` 的网站静态资源 `URL`、并确保其已填入「基本信息」页对应位置后，再执行上述发布配置文件到远程的操作。

### 维护配置

#### 1. 锁定配置

设定好当前阶段的相关配置后，可在「系统设置」页的「锁定管理」分组锁定所有配置，避免误触修改。锁定后如需修改，亦可解锁配置。

#### 2. 导入导出

若因网络问题无法通过云存储进行双端配置通信，管理员可在「系统设置」页的「导入导出」分组将配置文件导出并直接分发给成员。管理员亦可用导出的配置在其他设备上导入后继续操作。

#### 3. 设置密码

若需避免他人查看或更改信息，可在「系统设置」页的「密码管理」分组设置密码，设置后管理员端每次启动应用时需要输入密码验证。


#### 4. 效果检查

如需查看配置在成员端的呈现效果，管理员可在「系统设置」页的「模式管理」分组切换到成员模式来查看。请确保在切换前，将「基本信息」页的「成员可否切换模式」设为“允许”，以免无法切回。

## 发展成员操作指南

### 获取配置

首次使用时，需要获取所在支部的管理员配置文件 `admin_config.json`：

#### 方式一：通过 URL 同步

- 向支部管理员获取配置同步 URL
- 启动应用时选择「通过 URL 同步配置」
- 输入 URL 完成同步

#### 方式二：从本地导入

- 向支部管理员获取配置文件
- 启动应用时选择「从本地导入配置文件」
- 选择文件完成导入

只要获取到的配置文件中包含「配置同步URL」键值，此后应用启动时会自动从该 URL 同步最新配置。若遇网络问题无法在启动时自动同步，仍可在「系统设置」页的「支部配置管理」分组再次手动同步或本地导入。

### 完善材料

#### 1. 填写基本信息

「基本信息」页提供了「支部公共信息」和「个人基本信息」两个分组，成员可在前者中了解支部若干重要信息，可在后者中点击「编辑」填写个人信息，完成后点击「保存」。

#### 2. 填写模板字段

当需要填写待归档材料时，进入「模板列表」页，点击需要填写的模板，在「模板详情」页填写该模板的专有字段，点击「保存」。

管理员会根据需要设定若干字段的值，成员可根据字段旁的信息来填写：
- 已锁定：管理员已为该字段设置固定值，成员无需修改
- 待确认：管理员已为该字段填写相应值以用于提示，成员可根据自身情况完善信息或保持原值
- 无提示：管理员未配置该字段，成员需根据自身情况来填写

#### 3. 导出文档

- 方式一：单个导出——在模板详情页面，点击「导出」按钮，系统自动生成填充好信息的 `Word` 文档。如有需要，可在「系统设置」页中修改导出文件的保存位置。

- 方式二：批量导出——在「模板列表」页，灵活使用 `Ctrl` 或 `Shift` 键来勾选需要导出的模板。点击「批量导出」按钮即可生成相应文档。

### 维护档案

#### 1. 本地存档

当某份材料填写完成，且不需要再修改时，可以通过锁定操作来固定留存该材料文件的信息。

在「模板详情」页填写完所有字段，点击「锁定材料」按钮，锁定后该材料将变为只读状态。锁定材料后，成员可通过「档案图片管理」留存档案的实物图片，并可以删除或查看。

> [!IMPORTANT]
> - 配置快照机制仅作用于专有项，管理员关于基本信息的未来配置仍会影响成员端相应数据呈现，故强烈建议成员在每份材料归档后即锁定材料
> - 锁定操作不可撤销，请确保材料填写无误后再锁定
> - 根据相关法规，个人严禁擅自拍照留存党员档案，成员可留存本人会议照片、允许的资料等。

#### 2. 数据迁移

成员如需在其他设备上操作，可提前在「系统设置」页的「个人数据管理」分组中将个人数据文件 `member_info.json` 导出并发送至目标设备，在该设备上导入个人数据即可。

#### 3. 密码保护

进入「系统设置」页，在相应分组中设置密码保护个人档案数据，下次启动时需要输入密码。

## 高级功能

用户可根据需要按如下步骤在工具中添加并使用任何模板材料：

### 1. 定义字段规则文件 `fields_definition.json`

分析任务场景，确定个体基本项、整体基本项、模板常见项，并制定每项的规则。以就业类的表格材料为例，可能的字段规则定义如下：

```json
{
  "version": "1.0.0",
  "last_updated": "2026-01-01",
  "admin_fields": [
    {
      "group": "部门",
      "group_order": 1,
      "fields": [
        {"key": "部门名称", "type": "text", "required": true, "display": {"order": 1}},
        ......
      ]
    },
    ......
    {
      "group": "交互设置",
      "group_order": 4,
      "fields": [
        ...
      ]
    }
  ],
  "member_fields": [
    {
      "key": "行业", "type": "select", "required": true, "options": ["农业", "工业", "其他"], "display": {"label": "行业", "order": 2}
    },
    ......
  ],
  "template_fields": [
    {
      "key": "内容", "type": "textarea", "required": false, "match_keywords": ["内容"], "display": {"placeholder": "请输入内容", "rows": 5}
    },
    ......
    {
      "key": "其他", "type": "text", "required": false, "is_default": true, "display": {}
    }
  ]
}
```

### 2. 准备 Word 模板

准备 `.docx` 格式的 Word 模板文件。确保文件名不重复，推荐命名为 `template_(\d+)_(.+)\.docx`，且确保 `(\d+)` 不重复。根据上述字段规则在模板待填空处填入占位符语法：`{{ 字段名 }}`，占位符不要有标点，建议如需使用符号则仅用下划线“_”。示例如下：
```bash
┌──────────┬─────────────┐
│ 行业类型  │ {{行业}}    │
└──────────┴─────────────┘
```

### 3. 在工具中使用
- 将 `fields_definition.json` 放入 `resources/templates/` 目录
- 将 Word 文件放入 `resources/templates/` 目录
- 重启应用后即可使用新模板

## 常见问题 (FAQ)

### 管理员端相关

#### Q: 配置同步 URL 支持哪些服务？

A: 任何返回 JSON 内容的公开 URL 均可，例如：
- GitHub Gist（使用 Raw 链接）
- 云存储直链（如阿里云 OSS、腾讯云 COS）
- 自建服务器

### 成员端相关

#### Q: 什么是「锁定材料」？

A: 当某份材料填写完成且确认无误后，可以锁定该材料。锁定后：
- 该材料变为只读状态
- 无法再修改内容
- 防止误操作导致数据丢失

#### Q: 可以修改支部配置的信息吗？

A: 不可以。配置信息由管理员统一配置，成员只能查看不能修改。如需修改，请联系支部管理员开放「切换到管理员模式」的权限。

#### Q: 配置同步失败

A: 请检查：
1. URL 是否正确且可访问
2. 网络连接是否正常
3. URL 返回的是否为有效的 JSON

#### Q: 某些字段没有自动填充

A: 请确认：
1. 对应的基本信息是否已填写并保存
2. 模板中的占位符名称是否正确


### 通用

#### Q: 数据保存在哪里？

A: 保存在程序目录下的 `data/` 文件夹中：
- `admin_config.json` - 管理员配置
- `member_info.json` - 成员个人信息
- `system_settings.json` - 系统设置
- archive_images - 存档图片

#### Q: 如何备份数据？

A: 复制 `data/` 目录中的所有数据即可。

#### Q: 密码保护如何工作？

A:
- 设置密码后，数据会使用 AES-256 加密存储
- 每次启动程序需要输入密码验证
- 密码错误 3 次后程序自动退出

#### Q: 忘记密码怎么办？

A: 很遗憾，由于数据已加密，无法恢复。唯一的办法是删除 `data/` 目录下的对应文件重新开始。

#### Q: 数据会上传到互联网吗？

A: 程序仅会上传管理员配置文件 `admin_config.json` 到互联网，除此之外的任何数据均不会上传到互联网。

#### Q: 如何完全重置程序？

A: 删除以下目录/文件后重新启动：
- `data/` 目录
- `exports/` 目录
