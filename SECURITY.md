# 安全策略

## 支持的版本

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | :white_check_mark: 支持 |
| < 1.0 | :x: 不支持 |

## 报告漏洞

如果你发现了安全漏洞，请**不要**通过公开的 Issue 报告。

### 报告方式

1. 发送邮件至项目维护者（请在仓库主页查看联系方式）
2. 或通过 GitHub 的 [Security Advisories](../../security/advisories/new) 私密报告

### 报告内容

请在报告中包含以下信息：

- 漏洞类型（如：数据泄露、权限绕过、注入攻击等）
- 漏洞描述
- 复现步骤
- 影响范围
- 可能的修复建议（如有）

### 响应流程

1. **确认收到**：我们会在 48 小时内确认收到你的报告
2. **评估漏洞**：我们会评估漏洞的严重程度和影响范围
3. **修复开发**：开发并测试修复方案
4. **发布更新**：发布安全更新版本
5. **公开披露**：在修复发布后，我们会公开漏洞详情并致谢报告者

## 安全最佳实践

### 对于用户

- 定期更新到最新版本
- 设置强密码保护敏感数据
- 不要将包含敏感信息的配置文件上传到公开服务器
- 定期备份数据

### 对于开发者

- 不要在代码中硬编码敏感信息
- 使用环境变量或加密配置存储密钥
- 提交前检查是否意外包含敏感文件
- 遵循最小权限原则

## 安全特性

本项目已实现以下安全措施：

- **数据加密**：使用 Argon2 密钥派生 + AES-GCM 加密敏感数据
- **密码保护**：支持为管理员配置和成员信息设置密码
- **输入验证**：对用户输入进行格式验证（如身份证号、手机号）
- **安全传输**：配置同步使用 HTTPS 协议
