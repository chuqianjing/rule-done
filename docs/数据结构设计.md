# 数据结构设计文档

## 一、数据分类与存储位置

### 1.1 三类数据概览

| 数据类型 | 存储位置 | 可编辑性 | 权限控制 |
|---------|---------|---------|---------|
| 程序必要数据 | `resources/` | 只读（开发者） | 随程序分发 |
| 管理员配置数据 | `config/admin_config.json` | 可编辑（管理员） | 成员端只读 |
| 成员个人数据 | `data/member_info.json` | 可编辑（成员） | 仅成员本人 |

## 五、数据合并逻辑

### 5.1 数据源优先级
当生成 Word 文档时，数据合并顺序：
1. **管理员配置数据**（最高优先级，用于公共信息）
2. **成员基础信息**（个人信息）
3. **模板特有数据**（模板特定字段）

### 5.2 合并示例
```python
# 伪代码示例
def merge_data_for_template(template_id):
    merged_data = {}
    
    # 1. 加载管理员配置
    admin_config = load_admin_config()
    merged_data.update(extract_admin_fields(admin_config))
    
    # 2. 加载成员基础信息
    member_info = load_member_info()
    merged_data.update(member_info['basic_info'])
    
    # 3. 加载模板特有数据
    template_specific = member_info['template_data'].get(template_id, {})
    merged_data.update(template_specific)
    
    return merged_data
```

## 六、数据持久化方案

### 6.1 JSON 文件存储
- **格式**：UTF-8 编码的 JSON 文件
- **缩进**：2 个空格
- **换行**：LF（Unix 风格）

### 6.2 数据备份机制
```json
{
  "backup_enabled": true,
  "backup_strategy": {
    "auto_backup": true,
    "backup_on_save": true,
    "backup_interval_hours": 24,
    "max_backups": 10,
    "backup_path": "./backups"
  }
}
```

### 6.3 数据加密（可选）
```json
{
  "encryption": {
    "enabled": false,
    "algorithm": "AES-256",
    "key_derivation": "PBKDF2"
  }
}
```

## 七、数据迁移与版本管理

### 7.1 版本字段
所有数据文件都包含 `version` 字段，用于：
- 数据格式迁移
- 兼容性检查
- 升级处理

### 7.2 迁移策略
```python
# 伪代码
def migrate_data(data, from_version, to_version):
    if from_version == "1.0" and to_version == "1.1":
        # 执行迁移逻辑
        data = add_new_fields(data)
        data['version'] = "1.1"
    return data
```

## 八、数据完整性校验

### 8.1 校验规则
- 必填字段检查
- 数据类型验证
- 格式验证（日期、身份证号等）
- 逻辑关系验证（如转正时间不能早于入党时间）

### 8.2 校验结果存储
校验结果存储在 `member_info.json` 的 `validation_status` 字段中。
