# 配置同步功能说明

## 功能概述

系统提供了两种配置同步方式，帮助支部管理员统一管理配置信息：

1. **文件导入/导出**（离线方案）：管理员导出配置 JSON 文件，学生端导入使用
2. **联网自动同步**（在线方案）：管理员配置同步 URL，程序自动检查并更新配置

## 一、文件导入/导出功能

### 管理员端：导出配置

1. 在管理员配置页面，点击 **"导出配置"** 按钮
2. 选择保存位置和文件名（默认：`admin_config.json`）
3. 配置将导出为 JSON 文件，包含所有支部信息、公共字段等
4. 导出的配置文件可以分发给学生用户

**注意**：导出的配置不包含锁定状态等本地设置，适合分发给学生使用。

### 管理员端：导入配置

1. 在管理员配置页面，点击 **"导入配置"** 按钮
2. 选择要导入的 JSON 配置文件
3. 系统会：
   - 验证配置文件格式
   - 自动备份当前配置
   - 询问是否继续导入
   - 导入后保留本地锁定状态等设置
4. 导入成功后，表单会自动刷新显示新配置

### 学生端：导入支部配置

1. 在基本信息页面，点击 **"导入支部配置"** 按钮
2. 选择管理员提供的配置 JSON 文件
3. 系统会：
   - 验证配置文件格式
   - 自动备份当前配置（如果存在）
   - 导入配置并自动锁定（学生端只读）
   - 刷新显示新的支部信息
4. 导入成功后，页面会自动更新显示新的支部公共信息

## 二、联网自动同步功能

### 配置同步 URL

1. 在管理员配置页面的 **"系统设置"** 分组中，找到 **"配置同步 URL（可选）"** 字段
2. 填写配置文件的网络地址，例如：
   - GitHub Raw URL：`https://raw.githubusercontent.com/user/repo/branch/config.json`
   - 自建服务器：`https://example.com/api/config.json`
   - 其他支持 HTTP/HTTPS 的 JSON 文件地址
3. 保存配置

### 自动同步机制

程序在以下情况下会自动检查并同步配置：

1. **程序启动时**（仅管理员模式）：
   - 如果配置了同步 URL 且配置未锁定
   - 程序会在后台线程中检查远程配置
   - 如果远程配置更新，会自动下载并应用
   - 同步成功会显示提示信息

2. **配置加载时**（可选）：
   - 在 `ConfigManager.load_config(check_sync=True)` 时也会检查同步
   - 目前主要用于管理员模式

### 同步逻辑

1. **时间戳比较**：
   - 比较远程配置和本地配置的 `last_modified` 字段
   - 只有当远程配置更新时才同步
   - 如果时间戳相同或本地更新，则跳过同步

2. **配置验证**：
   - 验证远程配置包含必需字段（`branch_info`、`party_committee`、`common_fields`）
   - 验证 JSON 格式正确性

3. **备份机制**：
   - 同步前自动备份当前配置到 `.backup` 文件
   - 备份文件保存在 `config/` 目录

4. **配置合并**：
   - 保留本地设置（如锁定状态）
   - 更新配置数据（支部信息、公共字段等）
   - 记录同步时间和来源

### 强制同步

如果需要强制同步（忽略时间戳），可以在代码中调用：

```python
from src.utils.config_sync import ConfigSync
from src.data.config_manager import ConfigManager

config_manager = ConfigManager()
sync_manager = ConfigSync(config_manager)
success, message = sync_manager.check_and_sync(sync_url, force=True)
```

## 三、使用场景

### 场景 1：小规模使用（推荐文件导入）

1. 管理员在配置页面填写并保存配置
2. 点击"导出配置"，生成 `admin_config.json`
3. 通过 U盘、邮件、QQ 等方式分发给学生
4. 学生在基本信息页面点击"导入支部配置"，选择文件即可

**优点**：完全离线，隐私安全，操作简单

### 场景 2：大规模使用（推荐联网同步）

1. 管理员将配置导出为 JSON 文件
2. 将文件上传到 GitHub、云存储或自建服务器
3. 在管理员配置页面填写同步 URL
4. 保存并锁定配置
5. 学生端程序启动时自动检查并同步最新配置

**优点**：自动化，及时更新，统一管理

### 场景 3：混合使用

1. 管理员配置同步 URL（用于自动更新）
2. 同时提供文件导出功能（作为备用方案）
3. 学生可以选择：
   - 使用自动同步（需要网络）
   - 或手动导入文件（完全离线）

## 四、技术细节

### 配置文件格式

导出的配置文件格式：

```json
{
  "version": "1.0",
  "configured": true,
  "branch_info": {
    "branch_name": "支部名称",
    "branch_code": "001",
    "secretary": {
      "name": "书记姓名",
      "position": "党支部书记"
    }
  },
  "party_committee": {
    "name": "上级党委名称",
    "secretary": {
      "name": "党委书记姓名"
    }
  },
  "common_fields": {
    "school_name": "学校名称",
    "college_name": "学院名称"
  },
  "system_settings": {
    "export_path": "./exports",
    "config_sync_url": "https://example.com/config.json"
  },
  "date_format": {
    "format": "YYYY年MM月DD日"
  },
  "exported_at": "2026-01-24T12:00:00",
  "export_version": "1.0"
}
```

### 同步 URL 要求

1. **必须返回有效的 JSON 格式**
2. **建议支持 HTTP HEAD 请求**（用于快速检查）
3. **建议设置正确的 HTTP 头**：
   - `Content-Type: application/json`
   - `Last-Modified: <时间戳>`（可选，用于快速比较）

### 错误处理

- **网络错误**：同步失败不影响本地配置使用
- **格式错误**：验证失败会拒绝导入/同步
- **备份失败**：会尝试继续，但可能无法回滚

## 五、注意事项

1. **学生端配置锁定**：
   - 学生端导入的配置会自动锁定
   - 学生无法修改管理员配置的字段
   - 只有管理员模式才能解锁和修改

2. **配置版本兼容性**：
   - 建议保持配置版本一致
   - 不同版本的配置可能字段不同

3. **网络环境**：
   - 联网同步需要稳定的网络连接
   - 如果网络不稳定，建议使用文件导入方式

4. **安全性**：
   - 配置文件不包含学生个人信息
   - 仅包含支部公共信息
   - 建议通过安全渠道分发配置文件

## 六、常见问题

**Q: 学生端可以配置同步 URL 吗？**
A: 可以，但学生端导入的配置会自动锁定，无法修改。如果需要更新，需要重新导入或使用管理员模式。

**Q: 同步失败怎么办？**
A: 同步失败不会影响本地配置的使用。可以：
- 检查网络连接
- 检查同步 URL 是否正确
- 使用文件导入方式作为备用

**Q: 如何回滚到之前的配置？**
A: 系统会在同步/导入前自动备份配置，备份文件位于 `config/admin_config.json.backup`，可以手动恢复。

**Q: 多个学生可以共享同一个配置文件吗？**
A: 可以，配置文件只包含支部公共信息，不包含学生个人信息，可以安全共享。
