# 技术栈建议文档

## 一、推荐技术栈（Python + PyQt6）

### 1.1 核心框架

#### 桌面应用框架：PyQt6
**选择理由**：
- ✅ 跨平台支持（Windows / macOS / Linux）
- ✅ 成熟稳定，企业级应用广泛使用
- ✅ 丰富的 UI 组件
- ✅ 良好的文档和社区支持
- ✅ 可打包为独立 exe/app

**版本要求**：PyQt6 >= 6.6.0

#### 编程语言：Python 3.9+
**选择理由**：
- ✅ 语法简洁，开发效率高
- ✅ 丰富的第三方库生态
- ✅ 易于维护和扩展
- ✅ 跨平台兼容性好

### 1.2 Word 文档处理

#### 主要方案：docxtpl
**选择理由**：
- ✅ 基于 python-docx，功能强大
- ✅ 支持 Jinja2 模板语法
- ✅ 自动保持格式
- ✅ 支持复杂模板逻辑（条件、循环等）

**备选方案**：python-docx
- 如果不需要复杂模板功能，可使用纯 python-docx
- 更轻量级，但需要手动处理格式保持

### 1.3 数据存储

#### JSON 文件存储
**选择理由**：
- ✅ 简单易用，无需数据库
- ✅ 人类可读，便于调试
- ✅ 适合小规模数据
- ✅ 易于备份和迁移

**实现方式**：
- 使用 Python 标准库 `json`
- 自定义 `JSONStorage` 类封装读写操作

### 1.4 数据验证

#### 自定义验证器
**实现方式**：
- 使用 Python 标准库 `re` 进行正则验证
- 自定义 `Validators` 类
- 支持字段级和逻辑级验证

## 二、技术栈对比

### 2.1 桌面应用框架对比

| 框架 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **PyQt6** | 功能强大、跨平台、成熟稳定 | 体积较大 | ✅ **推荐** |
| Tkinter | Python 内置、轻量 | UI 较简陋 | 简单应用 |
| Electron | 现代化 UI、Web 技术栈 | 体积大、资源占用高 | Web 开发者 |
| .NET MAUI | Windows 原生、性能好 | 主要面向 Windows | Windows 专用 |

### 2.2 Word 处理库对比

| 库 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **docxtpl** | 模板语法强大、易用 | 依赖 python-docx | ✅ **推荐** |
| python-docx | 功能完整、稳定 | 需要手动处理格式 | 简单替换 |
| python-docx2txt | 仅支持读取 | 不支持写入 | 仅读取场景 |
| win32com | Windows 原生 | 仅 Windows、需要 Office | Windows + Office |

### 3.2 虚拟环境

**推荐使用 venv**：
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3.3 依赖管理

**requirements.txt** 已包含所有必需依赖：
```

python-dateutil>=2.8.2
```

## 四、打包发布方案

### 4.1 打包工具：PyInstaller

**安装**：
```bash
pip install pyinstaller
```

**打包命令**：
```bash
# Windows
pyinstaller --onefile --windowed --name="党员发展材料生成系统" main.py

# macOS
pyinstaller --onefile --windowed --name="党员发展材料生成系统" main.py

# 包含资源文件
pyinstaller --onefile --windowed --add-data "resources;resources" main.py
```

### 4.2 打包配置（.spec 文件）

创建 `party0101.spec`：
```python
# -*- mode: python ; coding: utf-8 -*-

block_cipher = None

a = Analysis(
    ['main.py'],
    pathex=[],
    binaries=[],
    datas=[('resources', 'resources')],
    hiddenimports=[],
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=[],
    win_no_prefer_redirects=False,
    win_private_assemblies=False,
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    a.binaries,
    a.zipfiles,
    a.datas,
    [],
    name='党员发展材料生成系统',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    upx_exclude=[],
    runtime_tmpdir=None,
    console=False,
    disable_windowed_traceback=False,
    argv_emulation=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
)
```

### 4.3 分发内容

**打包后的分发包应包含**：
1. 可执行文件（.exe / .app）
2. Word 模板文件（resources/templates/）
3. 配置文件模板（resources/）
4. 使用说明文档
5. 安装/运行说明

## 五、性能优化建议

### 5.1 模板缓存
- 模板文件加载后缓存，避免重复读取
- 使用单例模式管理模板管理器

### 5.2 数据加载优化
- 懒加载：只在需要时加载数据
- 增量保存：只保存修改的部分

### 5.3 UI 响应优化
- Word 生成使用异步处理，避免 UI 卡顿
- 长时间操作显示进度条

## 六、安全性考虑

### 6.1 数据安全
- ✅ 所有数据本地存储，不上传
- ✅ 敏感信息可加密存储（可选）
- ✅ 配置文件权限控制

### 6.2 代码安全
- ✅ 不包含硬编码的敏感信息
- ✅ 输入验证防止注入攻击
- ✅ 文件路径验证防止路径遍历
